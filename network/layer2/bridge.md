# bridge
ハブと違ってある程度信号の制御を行う賢いデバイス。

4つの種類が存在する

## セグメント
衝突ドメインのようなネットワークの区切りをセグメントという。

bridgeはセグメントとセグメントをつなぐ役割を担う。

※ハブでつなぐと１つのセグメントになってしまう。

## MACアドレス・フィルタリング
受け取ったフレームを通すかどうか、宛先MACアドレスを確認してフィルタリングしてくれる。

ただ、ブリッジを境目に通すか通さないかというフィルターでしかないため、

以下のようにセグメントとセグメントをつないで、BからCにフレームを送信した場合、CとDの両方にフレームが届きます。

※BからAに送信した時はフィルタリングされてC, Dには一切流れない。

```
A ┓        ┏ C
  ┣[bridge]┫
B ┛        ┗ D
```

### 利点
隣のセグメントには不要なフレームが流れにくくなるため、衝突の可能性が下がります。

その結果各セグメントの通信効率がアップします。（衝突時のJAM信号なども通さないため）

### 欠点
フレームを読み取る分、レイテンシが発生する。

あと、全体宛のブロードキャストは通ってしまうのはしかたないね。

### ソースルート・ブリッジ
IEEE802.5(トークンリング同士)をつなげるブリッジだそうです。特殊な部類らしい。まぁ高いもんね。

円状のルートの中でも早い方をルートとして選択する、賢い。

### トランスペアレント・ブリッジ
同じアクセス制御方式のセグメントをつなげるブリッジ。

e.g. イーサネットとイサーネット、FDDIとFDDIといった具合

### 変換ブリッジ
異なるアクセス制御方式のセグメントをつなげるブリッジ。

言葉でいうのは簡単だけど、そんな簡単に変換できるのだろうか...。

### エンキャプスレーション・ブリッジ
同じ制御方式のセグメント間をWANなどを通して中継するためのブリッジらしい。あまりでてこなさそうである。
